--- Groq → Ollama 마이그레이션 핵심 ---

schema_extractor.py:
  - OpenAI SDK → httpx 직접 호출 (Ollama 네이티브 API)
  - 이유: OpenAI 호환 API에서 Qwen3.5의 thinking 모드 비활성화 불가
  - Ollama 네이티브 API의 "think": false 옵션만 작동
  - 엔드포인트: POST http://localhost:11434/api/chat
  - timeout: 120초 (로컬 LLM은 Groq보다 느림)

.env:
  LLM_BASE_URL=http://localhost:11434/v1
  LLM_API_KEY=ollama
  MODEL=qwen3.5:27b-q8_0

--- 아키텍처 ---

서식은 로컬 컴퓨터에서 불러옴 (서버에 저장하지 않음):
  1. "서식 폴더 열기" → <input webkitdirectory> (브라우저 폴더 선택)
  2. JS가 로컬 폴더에서 schema.json + template.j2 읽음
  3. 서버 /api/session/upload-template에 업로드
  4. 서버 세션에 저장하여 LLM 프롬프트 + 렌더링에 사용

참고: showDirectoryPicker()는 HTTPS에서만 작동 → HTTP에서는 webkitdirectory 사용
참고: crypto.randomUUID()도 HTTPS 전용 → Math.random().toString(36) 사용

[7] 트러블슈팅 기록
================================================================================

1) Qwen3.5 thinking 모드 무한 생성
   증상: 12분+ 응답 없음, GPU 88% 점유
   원인: OpenAI 호환 API에서 think=false 미지원, /no_think 토큰 무효
   해결: Ollama 네이티브 API (POST /api/chat) + "think": false

2) crypto.randomUUID() 에러
   증상: JS 전체 동작 불능
   원인: HTTP 환경에서 Secure Context 필요 API 사용
   해결: Math.random().toString(36).slice(2,10)

3) showDirectoryPicker() 에러
   증상: 폴더 열기 실패
   원인: HTTPS 전용 API
   해결: <input webkitdirectory> 방식으로 대체

4) LibreOffice HTML→DOCX 실패
   증상: "no export filter" 에러
   해결: --convert-to "docx:Office Open XML Text" 필터 명시
   최종: python-docx 직접 생성으로 대체 (테이블 구조 보존)

5) DOCX 한글 깨짐 (네모 박스)
   원인: 맑은 고딕 폰트 미설치 (Linux)
   해결: Noto Sans CJK KR 폰트 사용 + w:eastAsia 설정

6) Playwright sync API in asyncio loop
   증상: PDF 변환 실패
   원인: FastAPI async 환경에서 sync_playwright 사용
   해결: async_playwright + await 사용

7) LibreOffice 동시 실행 충돌
   해결: -env:UserInstallation=file://{tmpdir} 임시 유저 프로필

8) 서버 reload 안 됨 (브라우저 캐시)
   해결: Ctrl+Shift+R 또는 ?v=N 쿼리 파라미터

--- UI 개선 ---
- 챗봇 인사말: "일반대화/서식활용" → "서식 선택 시 실시간 수정, 미선택 시 일반 챗봇"
- 헤더 글자 크기 1.5배 확대 (제목 text-2xl, 버튼 text-base, 뱃지 14px)
- A+/A- 글자 크기 버튼: 4단계 (12/14/16/18px), 입력창도 연동
- 시스템 메시지: 하늘색 테두리 박스 스타일 + 글자 크기 연동
- 서식 목록: 번호(#N) + 간단설명 추가
- 서식 목록/미리보기 박스 구분 (라벨 + 테두리)
- 서식 갱신 중: 빨간 반짝이는 뱃지 (toolbar에 표시, idle 시 안 보임)
- 서식 전환/OFF/폴더열기 시 확인 다이얼로그

--- 음성 입력 ---
- Web Speech API (SpeechRecognition) 기반
- 마이크 버튼: 입력창 옆, 클릭 시 빨간 펄스 애니메이션
- continuous + interimResults → 실시간 텍스트 표시
- Speech API 미지원 브라우저: 반투명 + 빨간 사선 + 비활성화
- HTTP 환경 제한: 마이크 권한 차단됨 (브라우저 보안 정책)
  → 해결법1: chrome://flags/#unsafely-treat-insecure-origin-as-secure 에

  → 해결법2: HTTPS 적용 (nginx + 자체 인증서)
